### Restrições críticas

- **`zod/v3`** — sempre importar de `'zod/v3'` nos schemas de formulário. `@hookform/resolvers@5.2.2` foi compilado contra zod v4.0.x e rejeita o v4.3.6 instalado por uma guarda de tipo literal no campo `_zod.version.minor`. A API gerada pelo Orval usa `import * as zod from 'zod'` (que é v4) — não tocar.
- **`acessToken`** — typo do backend mantido propositalmente no store e em toda a codebase; não corrigir para não divergir no regen.
- **`UserResponse.roles`** é `RoleResponse[]` (array de objetos `{role: string}`), não uma string. Admin check: `user?.roles?.some(r => r.role === 'ADMIN')`.
- **Next.js 16** tem breaking changes em relação ao treinamento do modelo — verificar `node_modules/next/dist/docs/` antes de assumir APIs.
- **NEXT_PUBLIC_API_URL** em `.env.local` = `http://localhost:8080` (backend Springdoc). Estava errado como `8081` — já corrigido.

### Decisões trancadas

1. Status: UI mantém 5 estados, API tem 4 — tradução em `src/lib/ticket-status.ts`.
2. `CANCELLED` é write-blocked (backend não suporta ainda).
3. DataTable genérico: `@tanstack/react-table` reutilizado em todos os recursos.
4. Refresh strategy: interceptor axios single-flight 401 → refresh → retry.
5. Route groups: `(auth)/` sem shell, `(app)/` com AppShell + guard.
6. Locale: pt-BR.

### O que está quebrado — 2 bugs ativos que devem ser corrigidos ANTES de continuar

#### Bug 1 — Runtime Error: `Cannot read properties of undefined (reading 'hasHydrated')`

**Arquivo:** `src/app/(app)/layout.tsx:15`

```tsx
// linha problemática:
const [hydrated, setHydrated] = useState(() =>
  useAuthStore.persist.hasHydrated(),  // <-- .persist é undefined durante SSR
)
```

**Causa:** Next.js 16 pre-renderiza componentes `'use client'` no servidor. Durante SSR, o middleware `persist` do zustand ainda não inicializou (depende de `localStorage`), então `useAuthStore.persist` é `undefined`.

**Fix:**

```tsx
const [hydrated, setHydrated] = useState(() =>
  useAuthStore.persist?.hasHydrated() ?? false,
)

useEffect(() => {
  if (hydrated) return
  return useAuthStore.persist?.onFinishHydration(() => setHydrated(true))
}, [hydrated])
```

#### Bug 2 — Console Error: `Encountered a script tag while rendering React component`

**Arquivo:** `src/app/providers.tsx:62` — `<ThemeProvider>` do `next-themes@0.4.6`

**Causa:** `next-themes` injeta uma `<script>` tag para prevenir FOUC. React 19 mudou o comportamento de `<script>` dentro de componentes — agora os trata como elementos DOM reais e loga erro de console.

**Fix (a investigar):**
- Opção A: Atualizar `next-themes` para versão que suporte React 19 (verificar changelog)
- Opção B: Adicionar `suppressHydrationWarning` no `<html>` em `layout.tsx`
- Opção C: Usar `scriptProps={{ nonce: undefined }}` no ThemeProvider

Este erro não quebra a app mas aparece em todo carregamento de página.
