## 2. CSS variables — important gotcha

`var(--wl-surface-1)` does **NOT** exist. The existing dashboard components use it incorrectly.

Available surface scale (`src/app/globals.css`):
| Variable | Light | Dark | Use for |
|---|---|---|---|
| `--wl-bg` | `#f5f3ef` | `#13110f` | Page background (darkest) |
| `--wl-surface` | `#ffffff` | `#1c1a17` | Card / panel background |
| `--wl-surface-2` | `#eeeae4` | `#242018` | Hover states, inputs |
| `--wl-border` | `#ddd9d2` | `#2e2b25` | Dividers |
| `--wl-text` | `#1a1714` | `#f0ebe3` | Body text |
| `--wl-text-muted` | `#7a7269` | `#7a7269` | Secondary text |

Blue accent for the right marketing panel = `var(--primary)` (oklch blue).

**Fix needed in existing components before implementing new ones:**
- `src/components/dashboard/status-donut.tsx` — `var(--wl-surface-1)` → `var(--wl-surface)`
- `src/components/dashboard/recent-activity.tsx` — same
